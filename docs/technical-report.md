## Technical Report: Job Application Agent

I built this project to solve a practical problem I kept running into during internship and full-time job applications: tailoring a resume and drafting a role-specific cover letter is repetitive, time-consuming, and hard to do consistently at high quality when deadlines are tight. I wanted a system that could take a real job posting URL, extract the description, compare it to my current resume, and then produce targeted edits that were specific enough to be useful but still grounded in my actual experience. Instead of building a one-shot chatbot, I intentionally built an agentic workflow that can complete this process as a sequence of tool-driven steps, because the task naturally depends on intermediate outputs (scraped job description, fit gaps, rewritten sections, and final cover letter).

The most important use case is accelerating real applications for data science, machine learning, and analytics roles while maintaining factual integrity. A second use case is educational: the app demonstrates how an LLM agent can coordinate multiple deterministic and model-based tools in a transparent loop, which is useful for class demos and technical interviews. A third use case is quality control. I added evaluative metrics so the system does not only generate content, but also critiques it by measuring whether the tailored resume actually improved keyword alignment (Fit Lift) and whether generated claims are supported by the original resume (Groundedness/Hallucination Rate). This turns the app from a pure generator into a lightweight generation-plus-evaluation pipeline.

From an architecture perspective, the app is built with Next.js App Router so one codebase can handle both the frontend and backend API route. The core agent is implemented using Vercel AI SDK 6 `ToolLoopAgent` with Anthropic `claude-sonnet-4-20250514` as the model. I defined tools in `lib/tools.ts` to keep responsibilities clear: `scrape_job_description`, `analyze_fit`, `rewrite_resume_section`, `draft_cover_letter`, `evaluate_fit_lift`, and `evaluate_groundedness`. The `scrape_job_description` tool includes robust fallback extraction logic for JavaScript-heavy job boards (including Workday), combining main-content parsing with meta-tag and JSON-LD extraction. The API route in `app/api/agent/route.ts` injects workflow constraints into each run so the agent follows the required order and returns complete outputs, including a full cover letter and evaluation results.

On the frontend (`app/page.tsx`), I designed the interface for practical use rather than just prototype aesthetics. The form accepts a job URL, resume baseline, and optional request note, then streams step-by-step output so I can see what the agent is doing in real time. I also added quality-of-life features for repeated usage: clear chat, local saved history (save/load/delete via localStorage), and a dedicated cover letter panel that surfaces the latest drafted letter directly. To reduce cognitive overload, verbose tool payloads are hidden by default behind a toggle, so the interface emphasizes the final deliverables first while still exposing internals when debugging is needed.

To use the system, I set `ANTHROPIC_API_KEY`, run the Next.js app, paste a job URL, and submit my resume text. The agent then scrapes and parses the posting, computes fit gaps, rewrites selected resume sections, drafts a full cover letter, and reports evaluative metrics. In deployment, I use Vercel with default Next.js settings and environment variables configured in project settings. Overall, this project demonstrates an end-to-end agentic application that is both useful for real job applications and technically meaningful as an example of tool orchestration, evaluation-aware generation, and user-centered output design.
